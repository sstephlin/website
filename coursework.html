Introduction to Computer Systems

Completed Projects:

  • Maze: a C programming assignment that generated mazes of any size by utilizing the drunken-walk algorithm and solved mazes of any size using a depth-first search algorithm. This project prioritized an understanding of structing C programs, algorithms, and concepts of pointers, structs, and bitwise operations.
  • Traps: utilized GDB to step through x86-64 assembly code in order to retrieve four hidden passwords needed to disarm terminally controlled traps.
  • Buffer: crafted exploit strings to create buffer overflow attacks to exploit a program, resulting in intended behaviors. Stepped through x86-64 assembly code to visualize the stack and registers of a given program. Additionalluy wrote machine code to exploit the program.
  • Shell 1: the first part of a program that models the behavior of a partial-functioning C shell. Handled reading and parsing of command lines, built-in commands, and input/output redirections.
  • Shell 2: the second part of a program that models the behavior of a partial-functioning C shell. Handled multiple jobs (foreground and background processes) and signal handling, masking, and updating.
  • Malloc: created a first fit dynamic storage allocator. Implemented malloc(), coalesce(), and realloc() from scratch by managing a doubly-linked free-list and resizing the heap.
  • Database: created a thread-safe server to manage a database (structured as a binary tree), handling concurrent clients who can add, delete, and query entries. Thread safely connected clients to a server, munipulated the database, and handled sendign signals, cancellation requests, and reaching end of file.

Data Structures and Algorithms: Course Overview

Completed Assignments:

  • HW1: an introduction to functional progamming:
    • HW1A: Understanding the basics of writing tests, working with records, and using maps and filters in Java.
    • HW1B: Learning recursive functions, annotating worst-case runtime, and implementing sorting algorithms.
    • HW1C: practice with tree-like data structures, conversion of lists to tree-like data structures, runtime annotation, and reasoning of design with tree-like data structures.
  • HW2 Understanding and Implementing Linked-Lists: practiced implementing mutable lists, understood how lists work in memory, analyzed runtime practiced testing interacting mehtods, and added private fields to increase performance speed.
  • HW3 Implementing Hashtables: practiced working with exceptions and curating a bredth of test-cases by applying newly-introduced hashtables to real world settings.
  • HW4 Introduction to Python: wrote comprehensive test cases based on function specification in order to debug existing code, learned list comprehension, applied Python sets, lists, and disctionaries, and used Python libraries to read command-line arguments and process files.
  • HW5 Sharing Information History: applied design and computation principles to a real-world situation under constrained resources, understood how to save and reload data, and used Python libraries to process files.
  • HW6 Applications: Solved 4 LeetCode-like problems under specific runtime constraints to apply data structures in new contexts and strengthen understanding of runtime and space complexity.

Completed Projects

Introduction to Object Oriented Programming: Course Overview

Completed Assignments:

  • Rattytouille: Understanding the basics of java, including compiling and running code, print lines, and defining and calling methods.
  • AndyBot: Covers the concepts of reading JavaDocs, parameters, and nesting methods through navigating AndyBot through a fixed maze.
  • Pong: Coding A simple rendition of Pong to cover concepts of parameters, containment, top-evel classes, instance variables, and writing methods.
  • Tic-Tac-Toe: Implemented a game of Tic-Tac-Toe by constucting a Tic-Tac-Toe board, controlling when players switch turns, and displaying updates. Applied concets of association, accessor and mutator mehtods, writing classes from scratch, and implementing interfaces.
  • Fruit Ninja: Applied concepts of polymorphism, inheritance, if-statements, and switch statements to model the game Fruit Ninja.
  • Cartoon: A creative project to code an animation using graphical user interfaces, graphical containment, layout management, complex association, sharing refereences, events, event-handling, JavaFX, and lamda expressions.
  • Doodle Jump: A partner project that implemented a game of Doodle Jump to cover topics of loops, arraylists, graphical and logical representation of programs.
  • Tetris: Applied concepts of nested loops, arrays, and writing algorithms to implement a simple game of Tetris.
  • Pacman: A final project that implemented a game board, had three different game modes, kept track of scores and lives, and used a BFS algorithm to ensure proper gameplay.
